Для обработки данных использовались Power Query и для создания мер Power Pivot


СКОЛЬКО ЧЕЛОВЕК ПРОЖИВАЕТ В КАЖДОМ ШТАТЕ.

- Для однозначной идетификации человека создаю поле "id_name" c уникальными значениями на основе конкатенации полей "first_name", "last_name", "phone1" (запрос в Power Query - "data").
- Загружаю запрос в Power Pivot и создаю меру "count name"
count name = COUNTROWS('data'[id_name]).
- Строю сводную таблицу на основаниии данных из таблицы в моделе данных Power Pivot - "data".
РЕЗУЛЬТАТ на листе "count_people".

СКОЛЬКО ИЗ НИХ С ОДИНАКОВЫМ ИМЕНЕМ.

- Загружаю таблицу "data" в Power Query (запрос - "duplicated_first_name").
- Сохраняю только строки по полю "first_name" имеющие дубликаты (меню "сократить строки" -> "сохранять дубликаты".
- Загружаю запрос в Power Pivot и создаю меру для подсчета уникальных значений в таблице "duplicated_first_name"
unique_name = DISTINCTCOUNT('duplicated_first_name'[first_name])
- Строю сводную таблицу только на основании меры unique_name.
- Для подсчета сколько раз повторяется каждое имя создаю меру 
duplicated_name = COUNTROWS('duplicated_first_name').
- Строю сводную на основании таблицы "duplicated_first_name" и меры "duplicated_name".
РЕЗУЛЬТАТ на листе "name_duplicated"

У КОГО УРОВЕНЬ ДОХОДА УПАЛ БОЛЕЕ ЧЕМ НА 50%.

- Загружаю таблицу "data" в Power Query (запрос - "income_name").
- Для однозначной идетификации человека создаю поле "id_name" c уникальными значениями на основе конкатенации полей "first_name", "last_name", "zip" (предполагаю если имя и фамилия совпадает, но zip разные, то это разные люди)
- создаю три меры:
income2015 = SUM([income 2015])
income2016 = SUM([income 2016])
%proc = [income2016] /[income2015] - 1
- Строю сводную на основании мер созданных выше
РЕЗУЛЬТАТ на листе "%proc_income".

В КАКИХ ШТАТАХ САМЫЙ БОЛЬШОЙ ПРИРОСТ ПО ДОХОДУ. 
    
- Из таблицы "data" делаю таблицу справочник с уникальным полем "zip" (ключ для объединения с таблицей "add" для определения штата по zip) и полем "state" (запрос - "id_state").
- Объединяю таблицы "id_state" и "add" (запрос - "merge_zip_state").
- Создаю меру для вычесления процента дохода
%proc_state = SUM([income 2016]) / SUM([income 2015] )- 1
- Строю сводную таблицу на основании данных таблицы "merge_zip_state" и меры %proc_state
РЕЗУЛЬТАТ на листе "%proc_state".

СКОЛЬКО ПОЧТОВЫХ АДРЕСОВ НАХОДЯТСЯ НА gmail, yahoo, hotmail, cox.

- Для отбора необходимых компаний загружаю таблицу "data" в Power Query (запрос - "email_count"). Используя инструмент Power Query "Разделить столбец по разделителю" из столбца "email" вычленяю название компании.
- Создаю меру для подсчета количества строк
cont_email = COUNTROWS('email_count')
